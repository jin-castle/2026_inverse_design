## 1. 진단 결과 요약

### 문제 A: 소스 스펙트럼 불일치 (즉시 수정 가능)

```
현재 코드:
  fcen = mean([fR, fG, fB]) = mean([1.432, 1.859, 2.222]) = 1.838
  df   = fB - fR = 0.790

결과:
  소스 진폭 @fB=2.222: Gaussian 꼬리 → flux ≈ 0
  소스 진폭 @fR=1.432: Gaussian 꼬리 → flux ≈ 0
  소스 진폭 @fG=1.859: 중심 근처 → flux 정상
```

**수정**: `fcen = (1/0.400 + 1/0.720)/2`, `df = 1/0.400 - 1/0.720`

### 문제 B: Si 단파장 흡수 (구조적 한계)

```
Si 복소 굴절률 (Palik):
  λ=698nm (R): n=4.147, k=0.001 → 흡수 거의 없음 ✓
  λ=538nm (G): n=4.448, k=0.010 → 약한 흡수 ✓
  λ=450nm (B): n=4.885, k=0.068 → 흡수 5% 이상 ⚠
  λ=400nm:     n=5.570, k=0.387 → 흡수 매우 강함 ✗

penetration depth (1/α = λ/(4πk)):
  λ=698nm: δ = 698/(4π×0.001) ≈ 55,600nm → 55μm 깊이까지 전파
  λ=538nm: δ = 538/(4π×0.010) ≈  4,280nm → 4.3μm
  λ=450nm: δ = 450/(4π×0.068) ≈    526nm → 0.5μm ← 라우터 두께 수준!
```

**B 채널(450nm)이 Si 기판에서 빠르게 흡수 → flux 모니터까지 도달 못함**

논문이 왜 잘 작동하는가?  
→ RCWA는 Si 흡수를 photodetector current로 직접 계산 (흡수 = 신호)  
→ MEEP FDTD는 flux로 측정 → 흡수된 빛은 측정 안 됨!

**이게 근본적인 방법론 차이**: 논문 OE = 흡수된 파워, MEEP = 투과된 flux

### 문제 C: 방법론 차이 (RCWA vs FDTD)

| 항목 | 논문 (RCWA) | 우리 (MEEP FDTD) |
|------|------------|-----------------|
| **효율 정의** | 포토디텍터 **흡수 파워** | 모니터 **투과 flux** |
| **Si 처리** | 흡수 = 신호 | 흡수 = 손실로 측정 |
| **최소 feature** | Fourier order 제한 | resolution 격자 제한 |
| **근접장** | 모든 Fourier order 포함 | 격자 크기까지만 |
| **계산 비용** | 단일 파장 O(N³) | 광대역 but 시간 O(t×N³) |

**핵심**: MEEP에서 올바른 효율 측정은 Si 내부 흡수를 직접 계산해야 함  
→ `dft_flux` 대신 `get_dft_array` + `|E|² × ε_i` 적분 필요

### 문제 E: Si 분산 모델

| 항목 | 논문 | 우리 |
|------|------|------|
| Si 모델 | 파장별 n+ik (측정값) | eps=19.8 고정 (D_cond@550nm) |
| 오차 @400nm | - | ε_r=30.9 vs 19.8 = 57% 오차! |
| 오차 @550nm | - | ε_r=19.8 vs 19.8 = OK |
| 오차 @700nm | - | ε_r=17.2 vs 19.8 = 13% 오차 |

단파장(400~500nm)에서 n이 실제보다 낮게 모델링됨 → 굴절/반사 오차

## 2. 수정 우선순위

```
P1 [즉시, 코드 3줄] 소스 스펙트럼:
   fcen = (1/0.400 + 1/0.720)/2  = 1.944
   df   = 1/0.400 - 1/0.720      = 1.111

P2 [즉시, 코드 ~20줄] 효율 측정 방법 변경:
   현재: DFT flux at z_mon (Si 내부 투과)
   수정: Si 내부 |E|² × Im(ε) 적분 = 흡수 파워
   
   MEEP 코드:
     near_fields = sim.get_dft_array(dft_fields, ...)  # E field
     P_abs = np.sum(np.abs(E)**2 * omega * eps_imag * dV)

P3 [단기, ~1시간] Layer thickness 파라메트릭 스캔:
   Lt in [1.0, 1.2, 1.5, 1.8, 2.0] μm
   → 논문 효율(76/72/81%)에 가장 가까운 Lt 찾기

P4 [단기, ~2시간] Si 분산 파장별 개별 시뮬:
   각 파장(R, G, B)마다 해당 파장의 n+ik로 개별 Si 설정
   → 3개 파장 독립 시뮬 후 결과 합산

P5 [중기, SimServer] Resolution 증가:
   현재 res=10~20 → 목표 res=100 (10nm 격자)
   필요 메모리: Nx×Ny×Nz = 100×100×350 = 3.5M voxel (충분)
   
P6 [중기] 저자 이메일:
   catrysse@stanford.edu

```

## 4. 결론: 왜 차이가 나는가

```
가장 큰 원인 (80% 기여):
  RCWA(논문) = 흡수 파워 계산
  FDTD(우리) = 투과 flux 계산
  → 같은 물리지만 다른 양을 측정하고 있었음!

나머지 원인 (20%):
  - Binary pattern 없음 (다른 local optimum)
  - Si 비분산 모델 (특히 B 채널)
  - Resolution 제한 (50nm feature → 2~3 voxel)
  - Layer thickness 불확실
```
